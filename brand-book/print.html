Brand Story

Where NICHE came from, and what it refuses to be.

Overview

What NICHE is not

Not a bootcamp (bootcamps optimize for speed and certificates). Not a university (universities optimize for credentials and prestige). Not a workshop series (workshops are episodic, not systemic). NICHE is a cohort institute — intensive, peer-based, outcome-oriented.

The anti-mainstream position

The anti-mainstream position

The copy 'Are you learning wrong way?' is not a rhetorical flourish — it is the brand's core thesis. Most creative education teaches technique divorced from judgment, market, and self-knowledge. NICHE teaches all three simultaneously.

Origin voice

Origin voice

The founding story should be told in first person, past tense, with specific detail. No 'we realized' vagueness — actual events, dates, and decisions.

Do / Don't

Every violation listed here has happened. Don't let it happen again.

These are not preferences — they are rules. Each violation destroys a property that makes the mark distinctive.

Don't — Stretch Never scale disproportionately. Lock aspect ratio always.
Don't — Rotate Never rotate the wordmark. The tilted N is intentional — adding rotation conflicts with it.
Don't — Recolor Only use the three approved colorways. No custom colors, no gradients.
Don't — Low Contrast Never place the wordmark on a background that fails 3:1 contrast minimum.
Don't — Drop Shadow The mark is flat. No shadows, glows, emboss effects, or dimensional treatments.
Don't — Remove Period
N

WebAIM 2024: 83.6% of home pages have detectable WCAG contrast failures. NICHE's red (#fe1d25) and warm gray (#cccecd) both fail body-text contrast on light backgrounds. This is documented and enforced — not ignored. Design systems that document known failures prevent them. Systems that don't document failures repeat them.

Hard Rules

Three prohibitions. No exceptions.

Red — display only

#fe1d25 is for display text only — ≥18pt regular or ≥14pt bold. Never use as a hyperlink color on light backgrounds. Never use for body text at any size.

Warm gray — decorative only

#cccecd is for dividers, disabled states, and decorative backgrounds. Never use as text color on any background. Never use as a background for text elements.

Green — dark background only for body

#779152 as body text is only approved on #252026. On #f2f1f0, use for large display only. For body text on light — use #4a5c33 (accessible shade).

Do
Use red for section numbers, eyebrow labels, pull-quote accents, CTAs. Let it punctuate — never let it flood.
Don't
Use red as a hyperlink color, as body copy color, or as a paragraph background on any white/off-white surface.
Do
Use #4a5c33 (accessible green shade) for any body copy that must appear on #f2f1f0 in green. It passes 7.2:1 — more than enough headroom.
Don't
Use the warm gray (#cccecd) as placeholder text or secondary text. It reads as broken UI. Use rgba(37,32,38,0.5) for muted text instead.

Five Motion Tokens

Authority: IBM Carbon Motion. Adapted for NICHE.

Five durations cover everything NICHE will need. Reference the CSS variable, never the millisecond value directly. When the system recalibrates, one change propagates everywhere.

Token Duration Bar Use
--motion-micro 80ms
Icon state changes, checkbox toggles, hover fills. Imperceptible transitions that prevent jarring state snaps.
--motion-quick 150ms
Button press, tooltip appear, tag hover, card background cross-fade. The primary interaction token.
--motion-base 280ms
Card entrance, drawer open, modal appear, accordion expand. System transitions that communicate structure.

Page transitions, hero text reveals (per word). These transitions carry narrative weight — they should feel considered.

Easing Curves

Three curves. Every animation uses one of these.

Linear easing is never approved. Everything enters, exits, or adjusts in place — each requires its own physics.

Entrance cubic-bezier(0, 0, 0.3, 1)

Fast start, slow settle. For all elements entering the viewport or appearing on screen. Feels decisive, not tentative.

Exit cubic-bezier(0.4, 0.14, 1, 1)

Slow start, fast end. For elements leaving the screen. Pulls away with intention — doesn't linger.

Standard cubic-bezier(0.2, 0, 0.38, 0.9)

For in-place state changes: color swaps, hover fills, tab selections. Balanced — not theatrical.

Three Motion Principles

Motion earns its place. It doesn't decorate.

1
Type-first reveal

Display text enters word-by-word — not letter-by-letter. Letter-by-letter is the startup cliché. 40ms stagger per word. Fraunces deserves to be read, not animated.

2
Red activates on intent

#fe1d25 appears in motion only on hover/active states. It punctuates; it never auto-decorates on scroll. Red that fires without user intent trains users to ignore it.

3
Program cards cross-fade

On hover: background cross-fades #f2f1f0 → #252026 at quick (150ms). Category tag scales 1.0 → 1.05 simultaneously. The card reacts to presence — not to arrival.

Prohibited Motion Effects

These effects are banned. Not discouraged — banned.

Every prohibited effect below was either a cliché in 2022 or creates accessibility harm. Neither is acceptable at NICHE.

Letter-by-letter reveal

The startup-pitch cliché. NICHE uses word-by-word only. Letter-by-letter feels performative, delays legibility, and fails in reduced-motion contexts.

Parallax scroll

Scroll-linked position parallax causes motion sickness (vestibular disorder). Violates WCAG 2.3.3. Not allowed anywhere in the NICHE web system.

Auto-playing video with motion

Video that plays automatically with significant motion must include a pause control. Autoplay with motion without pause = WCAG 2.2.2 failure.

Bounce easing

bounce-in, bounce-out — toy register. NICHE is editorial. Use the three approved curves. If it needs to bounce, the design is wrong.

Infinite loop animations

Spinning logos, pulsing badges, looping backgrounds. They fight for attention without earning it. Motion at NICHE is triggered by user intent, not ambient.

Flash / strobe

Three flashes per second or more causes seizures. WCAG 2.3.1. No flashing at any frequency in NICHE digital materials. No exceptions.

prefers-reduced-motion

Every animation must respect this query.

WCAG 2.3.3 (Level AAA) and 2.2.2 (Level A) both address motion harm. Users with vestibular disorders, migraines, or ADHD can configure their OS to signal reduced-motion preference. NICHE's CSS must honor this signal everywhere.

    --motion-slow:   0ms;

Alt-Text Writing Guide

Alt text is a design decision. Write it like one.

Alt text is read by screen readers, indexing bots, and users on broken connections. Bad alt text is worse than no alt text. NICHE's alt text must be as intentional as its copywriting.

Do — write this
  • Describe what matters for the context, not everything visible.
  • For photography: what is the person doing, what objects are present, what is the mood or setting.
  • For logos: include the word "logo", the brand name, and the variant (e.g. "on dark background").
  • For PDFs used as references: describe the content of the page clearly, including any text visible.
  • For decorative images (backgrounds, dividers): use alt="" to mark as decorative.
Don't — avoid this
  • alt="image" or alt="photo"
  • Starting with "Image of..." or "Picture of..."
  • Including file names: alt="NICHE-logo-v2-FINAL.png"
  • Repeating surrounding text verbatim in alt text
  • Keyword stuffing: alt="NICHE logo brand design school Bangkok creative"
Logo alt text pattern "NICHE [variant] wordmark on [background] — [one distinguishing characteristic]"
Example: "NICHE OPT 02 primary wordmark on dark background — bold rounded humanist with tilted N"
Photography alt text pattern "[Subject/person] [action verb] [object/material] [context if relevant]"
Example: "Person annotating architectural sketches at a desk with natural window light"
Brand mockup alt text "NICHE [touchpoint type]: [key visual and text elements visible]"
Example: "NICHE outdoor poster: 'Find your niche' in white Fraunces over workshop photography"
Max length 125 characters is the soft limit. Longer descriptions use aria-describedby or figcaption instead.
Decorative images Use alt="" (empty string). Do not omit the alt attribute — missing alt = a11y fail.
Instagram Alt-Text

IG has native alt text. Use it. Every post.

Instagram allows custom alt text per post. It is not auto-generated from the caption. Most brands ignore it. NICHE will not.

How to add alt text in IG
  1. Post creation screen → Advanced Settings (bottom of screen)
  2. Write alt text → Enter a description of the photo
  3. Write the description following the photography alt text pattern above
  4. Save. The description is stored with the post and read by screen readers.
Required for All IG posts with images. Stories (where native alt text is available). Carousels — alt text per slide.
Caption vs alt text The caption is for the sighted audience. Alt text is for users who can't see the image. They serve different purposes — don't copy-paste.
Focus States

Keyboard users must see where they are.

WCAG 2.4.7 (Level AA): Any keyboard-operable user interface has a mode of operation where the keyboard focus indicator is visible. The default browser focus ring is ugly but functional. NICHE's custom focus style must be equally visible, ideally more so. 

/* Global focus style — applied in base.css */
:focus-visible {
  outline: 2px solid #fe1d25;  /* brand red = high visibility */
  outline-offset: 3px;
  border-radius: 2px;
}

/* Never do this: */
/* :focus { outline: none; } — accessibility failure */
/* :focus-visible { outline: 0; } — accessibility failure */

/* For dark backgrounds: */
.section--dark :focus-visible,
.cover-hero :focus-visible {
  outline-color: #f2f1f0;  /* off-white — 14.18:1 on dark */
}
Do
Use :focus-visible (not :focus) to show focus rings only for keyboard navigation, not mouse clicks. Users who click don't need the ring — keyboard users do.
Don't
Remove focus outlines for aesthetic reasons. If the focus ring breaks a design, fix the design — not the accessibility.
Semantic HTML Guide

HTML structure is the accessibility foundation.

Screen readers navigate by heading structure, landmark regions, and semantic elements. Semantic HTML that is correct requires no additional ARIA. ARIA that patches incorrect HTML creates new problems.

Page structure Every page must have: one <h1>, one <main>, one <nav> with aria-label, one <header>.
Heading hierarchy Never skip levels (h1 → h3 without h2). The visual size is independent of the semantic level. Use CSS to adjust size.
Images Every <img> must have alt attribute. Decorative: alt="". Informative: meaningful description. Required by WCAG 1.1.1.
Buttons vs links Use <button> for actions. Use <a> for navigation. Never use <div onclick> for either.
Form labels Every <input> must have an associated <label>. Placeholder text is not a label.
Language All pages must have <html lang="en"> (or "th" for Thai pages). Required by WCAG 3.1.1.
Color as only indicator Never use color as the sole means of conveying information. The WCAG pass/fail table in §5 uses text badges, not just color — this is intentional.
