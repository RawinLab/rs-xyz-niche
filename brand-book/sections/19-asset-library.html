Section 19

Asset Library & File Naming

Where files live, what they're called, and how to find them.

Overview

File naming convention

Pattern: [brand]-[asset-type]-[variant]-[color]-[version].[ext]. Example: niche-logo-opt02-on-dark-v1.svg. No spaces. No special characters. Use hyphens. Version numbers always.

Logo package structure

Logo package structure

logos/wordmark/opt01/, logos/wordmark/opt02/, logos/wordmark/opt03/, logos/submark/. Each folder contains: .svg, .pdf, .eps, .png@2x. On-dark and on-light variants in separate files.

Figma source structure

Figma source structure

One Figma file per section: NICHE-Brand-System (components + tokens), NICHE-Mockups-Print, NICHE-Mockups-Digital-Social, NICHE-Templates-Courses. Share via Figma Community or team workspace — not as exported .fig files.

Photo library

Photo library

photos/mode01-faculty/, photos/mode02-lifestyle/, photos/mode03-workdocs/. Tagged by program: photos/mode02-lifestyle/creativity/. All delivered RAW + graded TIFF + web JPEG in separate folders.

Font license files

Font license files

fonts/bai-jamjuree/LICENSE-OFL.txt alongside the font files. When GT Sectra is licensed: fonts/gt-sectra/LICENSE-GRILLI.txt. Never distribute font files without the license file.
